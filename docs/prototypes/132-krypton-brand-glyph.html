Krypton Brand Glyph — workspace-footer anchor

Spec 132, revised. The app mark lives at the left of the global workspace footer (krypton-workspace-footer, position:fixed; bottom:0; z-index:9000) — always visible across every view and workspace, like the Apple mark at the left of the macOS menu bar.

Why not the harness empty state (v1 idea): the harness auto-adds a default Claude lane on start (acp-harness-view.ts:2718), so the empty state is a dead surface users almost never see. Rejected. The footer is a persistent HUD element, so the mark sits at normal chrome opacity — and DESIGN.md’s “no illustrations in empty states” rule no longer applies.
