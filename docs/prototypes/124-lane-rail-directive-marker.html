Spec 124 — Lane rail · two-line directive marker

Rail rows grow to two lines: line 1 keeps the existing dot/name/metrics layout; line 2 is the meta line. When a directive is bound, the meta line shows ◇ directive title in full so users see the directive name at a glance — not just an icon glyph. Lanes without a directive still get a meta line so the row height stays consistent: it shows the status label (idle, busy, needs permission, …). Pending changes on a busy lane (pendingDirectiveChange) turn the meta line amber.

Meta line rules (priority, top down)
1. If activeDirectiveId (or pendingDirectiveChange) → ◇ directive.title. Title falls back to directive.id when title is empty. Long titles ellipsize.
2. Else → status label (idle, busy, needs permission, awaiting peer, error).

Color rules
• Plain directive (idle or busy, no pending change): purple icon, neutral title, faint purple row tint.
• pendingDirectiveChange on a busy lane: amber icon + title + · next send hint; row gets a faint amber tint.
• Pending clear (pendingDirectiveChange = { directiveId: null }): same amber treatment + strike-through on the icon to signal "going away"; the title shown is still the currently bound directive.
Pending changes on an idle lane apply immediately, so the amber state only appears for busy/needs-permission/awaiting-peer lanes.
