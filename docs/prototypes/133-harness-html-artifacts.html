A · The artifact card in the transcriptships v1

When a lane calls artifact_register, a card lands in the transcript — its own kind, distinct from tool / memory / peer rows. It shows title, lane, size + short content hash, and the issued .krypton/ path, never the HTML. The card is the only ambient affordance; it does not auto-open. live · editable means the lane may iterate on the same file across turns.

B · Open from hint mode — f labels the cardships v1

With several artifacts in the log, f overlays a label on each artifact card (the existing hint label system). These are "open artifact" hints, not URL hints — they reuse the labelling machinery, not the webview-opening semantics of spec 102. Press the label → that file opens in the browser. Works for any card, not just the most recent.

C · How it lands — and why the transcript stays smallships v1

Path-handoff: artifact_new issues a path → the lane writes the file with its normal write tool → artifact_register validates & cards it. The HTML never travels through MCP. But the lane's write tool card would still dump the HTML as a diff — so artifact-path writes are redacted to path + bytes + hash. That redaction, not path-handoff alone, is what protects the tail-window transcript (spec 103).

D · Iteration — the artifact is a live fileships v1

"Make the winning option green and add a verdict row." The lane reads k3.html back and edits it with its ordinary edit tool — no full re-send. The same card updates in place: new size, new hash, a updated marker. Editing is expected behaviour, so the card never pretends the bytes are frozen.

E · Ephemeral by designships v1

Artifacts live under .krypton/artifacts/ (a self-ignoring .gitignore keeps them out of the project's git). They are swept when the harness tab closes, and stale leftovers from a crash are swept on next startup. Reopening a closed-session card just reports the file is gone — reload/bookmark is not a promise.

◇auth-storage options — scorecard Claude-1 swept ↵ unavailable

If you need it to last, it was never an artifact — it should be a committed file the lane wrote on purpose (e.g. docs/), which is a different, non-ephemeral act.
