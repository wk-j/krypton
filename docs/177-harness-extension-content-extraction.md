# Harness Extension — Obsidian-grade Content Extraction — Implementation Spec

> Status: Implemented
> Date: 2026-06-27
> Milestone: M-ACP — Harness convergence

## Problem

When a user invokes the browser extension on a page **without selecting text**, the
extension sends only the title + URL (doc 176, "Selection empty — fall back to page
title + URL only"). The lane must then re-fetch the URL server-side — which fails on
the pages users most want summarized (Reddit, YouTube, login-walled, JS-rendered SPAs,
paywalls, geo-blocked content). The browser already has the rendered, authenticated
page in front of the user; we should capture it there instead of asking the agent to
fetch it again.

## Solution

Add **client-side content extraction** to the extension using **Defuddle** (the
extraction + HTML→Markdown library that powers Obsidian Web Clipper). When the popup is
invoked with no selection, the extension injects a bundled Defuddle into the active tab
on demand, extracts the page's main content as **clean Markdown plus metadata** (title,
author, description), and ships that as the prompt body — so the lane never needs to
fetch. Selection still wins when present; extraction is the fallback (and an opt-in
`{page}` placeholder for power users).

This supersedes two deferrals in doc 176: "No persistent content script" stays true (we
inject on demand, not via a declared `content_scripts` match), but "full-DOM-to-markdown
capture — out of scope" is now in scope via this spec.

## Research

- **Defuddle API** (`new Defuddle(document, opts).parse()`): returns `{ content,
  title, author, description, domain, image, language, wordCount, schemaOrgData,
  metaTags, favicon }`. Three entry points: `defuddle` (core, **no dependencies**,
  browser-focused), **`defuddle/full`** (adds math + **Markdown conversion** — required
  for `{ markdown: true }`), `defuddle/node` (server DOM). Options of interest:
  `markdown: true`, `removeImages`, `contentSelector`. Same library + `/full` path that
  Obsidian Clipper uses for `createMarkdownContent()`.
- **MV3 injection.** Content scripts cannot `import` npm modules at runtime, so Defuddle
  must be **bundled into one self-contained file** (IIFE). The current extension ships
  raw ES modules with **no build step** — this spec introduces a minimal `esbuild`
  bundle step for the injected file only; popup/background/options stay plain ESM.
- **On-demand vs declared content script.** `chrome.scripting.executeScript({ files })`
  injects the bundle only when the user clicks the popup (keeps the `activeTab` +
  `scripting` model from doc 176; nothing runs on pages the user never invokes).
  Injected files and a follow-up `func` call share the same ISOLATED world, so the
  bundle can expose a global the `func` then calls.
- **Existing contract.** `actions.js::renderTemplate()` already substitutes
  `{selection}/{title}/{url}` and wraps the body in `"""…"""`; adding `{page}` and a
  selection→page fallback is localized. `popup.js::getContext()` is the single capture
  point.

## Prior Art

| Tool | Extraction technique | Notes |
|------|---------------------|-------|
| Obsidian Web Clipper | content script → **Defuddle** (`defuddle/full`) + DOMPurify | Extract main content + metadata + schema.org, convert to Markdown client-side; exactly this design |
| Readability-based clippers (Mercury, older clippers) | Mozilla **Readability** + **Turndown** | Two libraries (extract, then HTML→MD); Defuddle merges both and handles code/math/footnotes better |
| Firefox Reader Mode | Mozilla Readability | De-clutters in-page; no Markdown output |
| MarkDownload extension | Readability + Turndown | Popular MD clipper; same client-side capture principle |
| Raindrop / Pocket | Server-side fetch + parse | Re-fetches URL server-side — fails on auth/JS pages, the exact problem we avoid |

**Krypton delta** — we adopt the Obsidian-grade client-side path (Defuddle) for the same
reason: the browser already holds the authenticated, rendered DOM, so extraction never
hits anti-bot/login/JS-render failures. We diverge from Obsidian in scope: no template
engine, no highlights, no DOMPurify round-trip (we send Markdown text to a trusted local
lane, not re-render HTML), and injection is on-demand (activeTab) rather than a declared
content script.

## Design

### Extraction source (bundled, injected on demand)

```js
// content-extract.src.js  → bundled to dist/content.bundle.js (IIFE)
import Defuddle from 'defuddle/full';

// No length cap (Open Question #2 resolved): the full extracted page is sent so
// long-form articles arrive intact; a very large page produces a large prompt.
globalThis.__kryptonExtract = () => {
  const r = new Defuddle(document, { markdown: true, removeImages: true }).parse();
  return {
    markdown: r.content || '',
    title: r.title || document.title || '',
    author: r.author || '',
    description: r.description || '',
    wordCount: r.wordCount || 0,
  };
};
```

### Capture (popup, two-stage)

```js
async function getContext() {
  const [tab] = await chrome.tabs.query({ active: true, currentWindow: true });
  const base = { selection: '', page: '', title: tab.title || '', url: tab.url || '', author: '' };
  try {
    const [{ result: selection }] = await chrome.scripting.executeScript({
      target: { tabId: tab.id }, func: () => String(window.getSelection() || ''),
    });
    base.selection = selection || '';
    if (!base.selection) {
      await chrome.scripting.executeScript({ target: { tabId: tab.id }, files: ['dist/content.bundle.js'] });
      const [{ result: ex }] = await chrome.scripting.executeScript({
        target: { tabId: tab.id }, func: () => globalThis.__kryptonExtract(),
      });
      if (ex) { base.page = ex.markdown; base.author = ex.author; base.title = ex.title || base.title; }
    }
  } catch { /* chrome://, store, or extraction failure → URL-only as in doc 176 */ }
  return base;
}
```

### Template rendering (actions.js)

- New placeholders: `{page}` (extracted Markdown), `{author}`.
- Body precedence in built-in templates: `ctx.selection || ctx.page`; wrap in `"""…"""`;
  emit `(no content)` only when both are empty.
- `Source:` line gains author when present: `Source: {title}{ — author} — {url}`.
- No truncation marker — the full extracted page is always sent (Open Question #2).

### Data Flow

```
1. User clicks extension on a page (no text selected)
2. popup.getContext(): executeScript reads window.getSelection() → empty
3. popup injects dist/content.bundle.js into the tab (activeTab/scripting)
4. popup executeScript(func) → calls __kryptonExtract() → { markdown, title, author, … }
5. renderTemplate(action, ctx): body = ctx.page (Markdown), Source line with author
6. background controlOp('lane.send', { lane, text }) → control API → lane
7. Lane receives clean Markdown — no server-side fetch needed
```

### Build

`extension/package.json`:

```json
{
  "private": true,
  "scripts": { "build": "esbuild content-extract.src.js --bundle --format=iife --outfile=dist/content.bundle.js --minify" },
  "dependencies": { "defuddle": "^0.x" },
  "devDependencies": { "esbuild": "^0.x" }
}
```

`make extension` runs `npm --prefix extension ci && npm --prefix extension run build`;
`make install` invokes it so the bundle exists before load-unpacked.

## Edge Cases

- **Restricted page** (`chrome://`, Web Store, PDF viewer) — `executeScript` throws →
  fall back to URL-only (unchanged doc-176 behavior).
- **Defuddle finds no article** (`content` empty — e.g. a pure app shell) — `page` is
  empty; body falls back to `(no content)` + Source line; lane still gets the URL.
- **Huge page** — sent in full (no cap, Open Question #2); a very large page yields a
  correspondingly large prompt. Accepted trade-off for completeness on long articles.
- **Selection present** — extraction is skipped entirely (selection wins; no injection).
- **Bundle missing** (forgot `npm run build`) — `executeScript({files})` throws a clear
  "Could not load file" error; README + `make` make this the happy path.
- **SPA / late-rendered content** — capture runs against the live DOM at click time, so
  content already rendered is captured; we do not wait for further async loads.

## Open Questions

_Recommended defaults below; confirm or adjust at approval:_

1. **Extract only when selection is empty, or always offer both?** Recommendation:
   selection wins, page is the fallback (keeps payloads tight and respects that a
   selection is precise user intent). Power users can use the `{page}` placeholder in a
   custom action to force full-page capture even with a selection. Confirm?
2. **Length cap — RESOLVED: no cap.** v1 first hardcoded `MAX_CHARS = 12000`, but real
   use on long-form articles (e.g. the GBrain piece) hit the limit, so the cap was
   removed — the full extracted page is always sent. Trade-off accepted: a very large
   page produces a large/expensive prompt. (A configurable Options-page cap remains a
   possible future refinement.)
3. **Build tooling = esbuild** (single-file IIFE, ~one line). Alternative: reuse the
   frontend's Vite. Recommendation: esbuild — lighter for one bundle, keeps the
   extension self-contained from the Tauri frontend deps. Confirm?
4. **Markdown vs cleaned HTML** — send Markdown (`markdown: true`). Confirm (vs sending
   cleaned HTML, which is noisier for an LLM)?

## Update — async parse (doc 179)

`__kryptonExtract` now awaits Defuddle's `parseAsync()` rather than the sync
`parse()`. The return shape is unchanged, but the async path lets Defuddle's
site-specific extractors run their async work — notably the **YouTube
transcript** extractor (InnerTube fetch, DOM-panel fallback). `popup.js`'s
`executeScript({func})` already returns the call result, which MV3 awaits when
it is a Promise. See `docs/179-youtube-transcript-extraction.md`.

## Out of Scope

- DOMPurify sanitization round-trip — we send Markdown text to a trusted local lane, not
  re-rendered HTML.
- Highlights / annotations capture (Obsidian Clipper feature).
- Screenshot / image capture; PDF text extraction.
- Firefox / Safari packaging (Chrome/Chromium family only, per doc 176).
- A declared `content_scripts` match (`<all_urls>`) — injection stays on-demand.
- Any new control operation — still rides `lane.send` / `lane.list`.
