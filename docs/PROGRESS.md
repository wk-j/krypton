## Recent Landings

- **Weight-dynamic attention gauge (spec 138)** — The workspace-footer attention chip (spec 128) was flat amber `N attention` — count only, weight-blind. It is now **weight-dynamic while staying motionless** (honoring the spec-128 / ADR-0001 static-by-design rule): the chip's colour tracks the **heaviest open reversibility tier** (cyan `reversible` → amber `costly` → red `irreversible`, reusing the triage-overlay palette) and a **pip strip** encodes the count (1–6 lit bars, label `6+` past the cap). `AttentionTriageStore.openItems()` is already sorted by reversibility descending, so element 0 is the max tier — no new computation. The `system:attention` ViewBus signal gained `maxReversibility: AttentionTier | null` (self-contained union in `view-bus-types.ts`); `acp-harness-view` publishes it (renamed `publishAttention`, dedupes on count **and** tier so a tier change at the same count still re-publishes); `WorkspaceFooter.attentionBySource` now holds `{count, tier}` per source, summing counts and max'ing tiers across harness tabs (one red item anywhere ⇒ red chip). No motion/pulse (rejected variant), no per-tier breakdown (only the max tier is carried). Visual chosen from an interactive prototype (variant D). See `docs/138-attention-gauge-weight-dynamic.md`.

- **ACP Harness agent image rendering** — agent-emitted markdown images now display in the transcript instead of broken-image boxes. Agents that generate images (e.g. Grok's `image_gen`, which writes a JPEG under `~/.grok/sessions/<enc-dir>/<id>/images/N.jpg` and emits `![alt](/abs/path.jpg)`) reference them by bare absolute filesystem path; the webview origin can't fetch raw FS paths, so they rendered as `?` placeholders. New `resolveLocalImageSrcs(root, cwd)` (next to the spec-117 `sanitizeSrc` helpers) walks `<img[src]>` in a rendered body and rewrites local sources to loadable asset URLs via `convertFileSrc()` (asset protocol already enabled in `tauri.conf.json` with `$HOME/**` scope) — a leading `/` is treated as a **true absolute path** (agent output), not cwd-root-relative as in the markdown viewer; relative paths resolve against the lane's project dir; remote/`data:`/already-resolved schemes are skipped so the pass is idempotent across re-renders. Wired at all three markdown→DOM sites: seal branch A (live foreground body, never re-rendered post-seal), seal branch B (offscreen background capture), and the `renderTranscriptItem` cold-load consumer (now threaded a `projectDir` param). A failed asset load swaps in a styled `IMG BREACH // <path>` notice; `.acp-harness__msg-body--markdown img` is capped to the column width. Verified the on-disk path + format with the live Grok-1 lane. See `docs/72-acp-harness-view.md`.

- **Markdown viewer — search, heading hints, image fix, focus indicator, re-select guard (spec 137)** — Five enhancements to the markdown viewer. (1) **In-doc search**: `/` in the preview opens a transient bottom HUD that wraps case-insensitive matches in `<mark>` (skipping `pre`/`code`, capped at 500), `n`/`N` (or `Enter`/`Shift+Enter`) cycle with a neon flash + `pos/total` counter, `Esc`/`q` clears and unwraps. (2) **Heading hints**: `H` reuses the link-hint label engine over `h1`–`h6` to jump+scroll — the engine was generalized to `enterHintMode(targets, onPick)` shared by links (`o`/`;`) and headings. (3) **Image fix**: relative `<img src>` is resolved against the current file's directory and loaded via `convertFileSrc()` (asset protocol enabled in `tauri.conf.json` with a `$HOME/**` scope — raw `file://` is blocked cross-origin); a failed load swaps in a styled `IMG BREACH // <path>` notice. (4) **Focus indicator**: the focused panel now shows a 3px glowing accent edge bar and a brightened, glowing header while the unfocused header dims — pure CSS on the existing `--panel--focused` class. (5) **Re-select guard**: `loadFile` early-returns when the file is already shown (re-selecting just moves focus to the preview without re-fetch/re-animate/scroll-reset); `r` passes `force` to still re-read from disk. Designed with peer input from Grok-1. See `docs/137-markdown-viewer-search-hints-images.md`.

- **ACP Harness prompt queue (spec 136)** — Typing a prompt while a lane is busy no longer flashes "lane busy" and discards it. `submitActiveLane` now captures the prompt into a per-lane FIFO `queuedPrompts` (text + a frozen `StagedImage` snapshot + mention targets resolved at enqueue via `parseMentionFanOut`), capped at `PROMPT_QUEUE_MAX = 10`. The agent-dispatch back-half was extracted into `sendUserPrompt(lane, text, images, opts)` — shared by the immediate submit and the queued drain; it never clears the live draft, so draining a queued prompt can't wipe what the user is typing (the immediate caller clears its own draft). `finishTurn` resets the turn pointers **before** its draining status transition (prerequisite fix: `setLaneStatus(idle)` synchronously drains peer mail re-entrantly, and the old tail-cleanup clobbered that new turn's state), then schedules `maybeDrainPromptQueue` via `queueMicrotask` to read the settled status — **peer mail wins** (it flips the lane back to busy first; the user queue holds, shown as `held behind lane mail`). A consumed-but-undelivered drain (a queued `@mention` whose target closed before drain) emits a `queued prompt not sent` row **and re-arms the drain** so the rest of the queue doesn't stall. Per-item keyboard management via `#unqueue [N]` / `#queue clear` / `#queue edit N` (strict `/^[1-9]\d*$/` index, command text consumed on every branch, `edit` guards staged images not draft text, re-sends to the tail); `#cancel`/`Ctrl+C` still clears the queue. The queue renders in a new bottom-anchored right-rail slot (`data-slot="queue"`, `flex-shrink:0` + capped height so plan/peek absorb the squeeze) — numbered drain-order rows, `▸` next marker, `→lane`/`img×N` tags — shown independently of the lane peek, hidden in zen; the composer chip appends `· N queued`. Reviewed adversarially by Codex-1 (×2), Grok-1, Claude-2 — 22 findings, all adopted. See `docs/136-acp-harness-prompt-queue.md`.

- **ACP Harness HTML artifacts (spec 133)** — A project-agnostic harness primitive letting a lane hand the user a rich, iteratively-editable HTML view. Path-handoff: a lane calls `artifact_new { title }` on `krypton-harness-bus`, the Rust hook server allocates an id and issues a destination path at `<cwd>/.krypton/artifacts/<harnessId>/<laneLabel>/<id>.html` (creating a self-ignoring `.krypton/artifacts/.gitignore`, failing closed if it can't), the lane writes the file with its normal file tool, then calls `artifact_register { id }`. Rust validates the file (rejects symlinks in any path component, hardlinks, non-regular files, wrong basename, over-cap size) and records size+sha256; `AcpHarnessView` raises a hintable card. The HTML never travels through MCP, and artifact-path write/edit cards are **redacted to path+bytes+hash** so HTML never enters the transcript model (the real spec-103 fix). Issued-path write auto-approval is a **separate** path/registry-keyed mechanism (not the spec-96 server-marker detector) keyed on path + issued-id + same lane. The card opens in the OS browser (ADR 0002) via `open_url(file://…)` from a per-card "open artifact" hint (`f` in command mode → label). Artifacts are live editable files: the harness re-stats on each observed edit; pending artifacts carry a write grant and are auto-cancelled at turn-end; everything is swept on harness close + a startup stale-sweep. Opt-in only (conservative-propensity guard in the tool description + lane-context stub). See `docs/133-harness-html-artifacts.md`, `docs/adr/0002-html-artifacts-open-in-os-browser.md`, and `docs/prototypes/133-harness-html-artifacts.html`.

- **ACP Harness directive-bound triage grant (spec 129)** — Made the spec-128 triage equip sourceable from a directive instead of only the manual `Leader '` toggle, which solves the "equip mid-session does nothing" trap (ACP clients fetch `tools/list` once at session start). Boolean over a general tool-grant list (deferred until ≥2 gated tools); mid-session reassign updates call-time auth + UI but not tool advertisement (documented caveat — a real fix needs MCP `tools/list_changed`). - **ACP Harness attention triage (spec 128)** — A per-lane opt-in router that surfaces the *decisions that need the human*, not lane activity. An equipped lane self-reports at end-of-turn via a new `attention_flag` MCP tool (`question` / `chosen` / `rationale` / `traded_off` / `uncertainty` / `reversibility`); the harness validates a presence floor (non-empty `traded_off`, non-blank `uncertainty`), assembles git blast-radius (diffstat) like a review packet, and inserts a `JudgementItem` into a **demand queue** (`AttentionTriageStore`) ranked by reversibility (irreversible > costly > reversible, ties oldest-first — `lane peek heat` is deliberately not consulted). The only ambient signal is a static **backpressure gauge** (`◆ N`) in the command center, hidden at zero. `Leader ;` summons a summon-on-demand overlay that shows one ranked card at a time, full-width (always shows traded-off + uncertainty, never collapsed; header shows the `2 / 5` cursor position); `j`/`k` page prev/next through the queue, `a` approves (bookkeeping only — no lane effect), `r` redirects (one-line input → `coordinator.deliverRedirect` injects on the lane's next idle), `o`/`Enter` digs into the lane transcript, `Esc` closes. A lane's `attention_resolve` demotes its own item to the **silent pile** (never deleted; terminal human status wins). Per-equipped-lane `LaneTriageStats` track flagged-vs-silent turns (busy→idle) for the ADR-0001 audit, shown in each card header. Equip is a runtime per-lane toggle (`Leader '`), mirrored into the hook server (`acp_set_lane_triage_equipped`) so `tools/list` advertises the tools only to equipped lanes; the call-time dispatch re-checks and is authoritative. Non-blocking, summon-on-demand, no auto-approval. Superseded for tool exposure by spec 130; see `docs/128-attention-triage.md` and `docs/130-default-attention-triage.md`.
